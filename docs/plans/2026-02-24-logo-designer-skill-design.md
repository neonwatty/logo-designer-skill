# Logo Designer Skill — Design Document

**Date:** 2026-02-24
**Status:** Approved

## Overview

A Claude Code plugin for iterative logo design using SVG. Claude writes SVG markup directly, generates a preview HTML for side-by-side comparison, and exports final logos to PNG at standard sizes.

## Phases

### 1. Interview
Claude asks 3-5 questions about the brand/project:
- Name
- Purpose/vibe
- Color preferences (or "surprise me")
- Style leanings (minimal, playful, geometric, etc.)
- Icon concept vs. wordmark vs. combination
- Canvas format (512x512 icon or 1024x512 horizontal)

### 2. Explore
- Generate 3-5 distinct SVG concepts
- Save to `logos/concepts/concept-1.svg`, `concept-2.svg`, etc.
- Generate `logos/preview.html` showing all concepts side-by-side
- User opens preview.html in browser, picks a direction

### 3. Refine
- Iterate on the chosen concept
- Save each iteration as `logos/iterations/iteration-N.svg`
- Regenerate preview.html to show latest iterations for comparison
- Continue as many rounds as needed

### 4. Export
- User says "export" or "I'm happy with this"
- Run bundled export script to produce PNGs at standard sizes
- Copy final SVG to `logos/export/logo.svg`

## Preview HTML

Self-contained HTML file (no external dependencies) that:
- Shows SVGs via relative `<img>` tags (works with `file://`)
- Grid layout of all current SVGs
- Dark/light background toggle
- Labels each card with filename
- Regenerated by Claude on every iteration — user refreshes browser

## SVG Conventions

- **viewBox-based sizing** — `viewBox="0 0 512 512"` (or appropriate ratio), no fixed width/height
- **No external dependencies** — no linked fonts, images, or `<use>` references
- **Web-safe fonts or paths** — system fonts or text converted to `<path>` elements
- **Meaningful structure** — `<g>` groups with IDs like `id="icon"`, `id="wordmark"`
- **Flat color palette** — solid fills by default, gradients only when requested
- **Standard canvas** — 512x512 for icons, 1024x512 for horizontal wordmarks

## PNG Export

**Primary tool:** `resvg` via `npx @aspect-build/resvg`

**Fallback chain:**
1. `resvg` (via npx)
2. `sharp` (via Node script)
3. `inkscape --export-type=png`
4. `rsvg-convert` (via librsvg)

**Bundled script:** `scripts/export.sh`
- Detects which tool is available
- Takes final SVG path as input
- Exports to sizes: 16, 32, 48, 192, 512, 1024, 2048
- Writes to `logos/export/`
- Reports error with install instructions if no tool found
